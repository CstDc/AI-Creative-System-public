# 新一期节目创建流程

版本：1.0
所属系统：圣经夜话公开版 AI 项目运行规范

## 一、完整流程

每一期节目必须经过五个阶段：

```
主题策划 → 内容创作 → 声音优化 → 审核发布 → 版本归档
```

## 二、第一阶段：主题策划

**使用窗口**：01_策划研究窗口

**输入**：可来自创作者提出的想法、听众反馈、现实生活观察、社会议题、圣经阅读中的发现。

**工作内容**：使用 01_主题选择工具、02_生命问题挖掘工具、03_经文匹配工具。

**输出**：《第XX期节目策划案》，包含节目主题、核心生命问题、听众入口、经文方向、内容目标。

## 三、第二阶段：内容创作

**使用窗口**：02_内容创作窗口

**输入**：《第XX期节目策划案》

**工作内容**：使用 04_播客结构设计模板、05_初稿生成流程。确认是否围绕一个生命问题、是否自然进入圣经、是否保持公开版表达方式。

**输出**：《第XX期文字初稿》

## 四、第三阶段：声音优化

**使用窗口**：03_声音优化窗口

**输入**：《第XX期文字初稿》

**工作内容**：使用 06_声音化转换流程，进行口语化调整、节奏调整、留白设计、情绪温度调整。

**输出**：《第XX期播客稿》

## 五、第四阶段：审核发布

**使用窗口**：04_审核发布窗口

**输入**：《第XX期播客稿》

**工作内容**：按照 06_审核标准 进行内容审核、神学审核、生命连接审核、最终发布检查。

**输出**：
- 通过 → 进入发布
- 修改后通过 → 返回对应窗口调整
- 不通过 → 重新进行相关阶段

## 六、第五阶段：版本归档

每期节目发布后建立归档目录：

```
第XX期_主题名称/
├── 01_策划案
├── 02_文字初稿
├── 03_播客稿
├── 04_审核记录
└── 05_最终版本
```

## 七、重要规则

1. 不跳步骤：不能没有策划案直接写稿，不能没有审核直接发布
2. 每一期只解决一个核心问题
3. 保留创作过程，不只保存最终稿（修改过程本身也是系统资产）
4. 每期结束后记录经验（哪些地方有效、哪些需要优化、AI 哪些地方理解错误）
