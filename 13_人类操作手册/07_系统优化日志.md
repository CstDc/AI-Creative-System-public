# 系统优化日志

用途：记录每期节目创作后的真实反馈，由创作者与系统优化者（AI）协作填写。
格式：日期 | 期数 | 反馈类型 | 具体问题 | 路由到

反馈类型分为两类：
- **有效**：哪个瞬间、哪种表达、哪个结构参数被验证为有效，应沉淀为案例或补入规则
- **需优化**：哪里不对劲、节奏偏快/慢、表达太远/太近、经文位置不合适，应修正对应规则

路由原则：
- 结构性经验（开头方式、推进节奏、结尾设计）→ 07_成功案例 或 08_失败案例
- 具体措辞/比喻/画面/节奏微调 → 05_声音表达
- 主题方向/生命问题挖掘 → 01_定位 / 02_听众
- 创作手法/结构 → 03_创作原则 / 04_内容结构
- 审核漏检 → 06_审核标准
